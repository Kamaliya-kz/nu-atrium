# Атриум · Комфорт сейчас

Full-stack сервис для мониторинга комфорта в атриуме Nazarbayev University.
Code Girl Summer 2026 Hackathon.

## Стек
- Frontend: React + TypeScript (`frontend/App.tsx`, `frontend/App.css`)
- Backend: Python + FastAPI (`backend/main.py`)
- База данных: SQLite через SQLAlchemy ORM
- Обмен данными: REST API / JSON

## Запуск backend

```bash
cd backend
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Сервер: http://localhost:8000, документация: http://localhost:8000/docs

Чтобы использовать реальные данные, положите файл `result.json`
(экспорт Telegram-канала) в папку `backend/` рядом с `main.py` **до первого
запуска** — импорт происходит автоматически при старте, если база пустая.
Если `result.json` отсутствует, backend генерирует демонстрационные данные
за 4 дня, чтобы проект можно было показать сразу.

## Запуск frontend

Файлы `App.tsx` и `App.css` рассчитаны на проект, созданный через Vite:

```bash
npm create vite@latest frontend-app -- --template react-ts
cd frontend-app
# скопируйте App.tsx и App.css в src/, заменив стандартные файлы
npm install
npm run dev
```

По умолчанию frontend обращается к `http://localhost:8000` (константа
`API_BASE` в начале `App.tsx`) — при деплое backend на Railway измените
это значение на публичный URL backend'а.

## Правила расчёта показателей

**Comfort score (0–100)**, для каждого измерения в атриуме:

```
score = 100
      − |температура − 22| × 4      (штраф за отклонение от идеальных 22°C)
      − штраф за шум:   тихо=0, средне=8, шумно=20
      − штраф за свет:  средне=0, темно=6, ярко=5
```

Итоговое значение ограничивается диапазоном [0, 100]. Это внутренняя
эвристика приложения, а не официальная санитарная норма.

**Текстовый вердикт** («Комфортно», «Жарко», «Очень жарко», «Подходит для
учёбы» и т.д.) вычисляется по температуре и уровню шума — см. функцию
`verdict_text()` в `main.py`.

**Лучший период для учёбы** — первый час дня, где шум = «тихо» и
температура находится в диапазоне 19–24°C.

**Разница атриум/улица** — средняя разница температур по совпадающим часам
измерений внутри и снаружи за выбранный день.

## Реализованные обязательные функции
- Импорт `result.json` → SQLite через ORM (или генерация демо-данных)
- Текущее состояние атриума с текстовым вердиктом и comfort score
- История измерений: фильтры по дате, месту, шуму, освещению, диапазону
  температур; сортировка по времени и температуре
- Аналитика: min/max/avg температура, самое прохладное/жаркое/тихое время,
  лучший период для учёбы, количество некомфортных измерений, разница
  атриум/улица, средний comfort score
- Отчёты пользователей: полный CRUD (создание, просмотр, изменение,
  удаление), поле `status` (open/resolved)
- REST API со всеми обязательными маршрutами, валидацией и корректными
  HTTP-кодами
- Frontend: состояния загрузки, ошибки, пустого результата и успешного
  действия (toast); адаптивная вёрстка

## Реализованные бонусные функции
- Мини-календарь с подсветкой дней, за которые есть данные
- Собственная формула comfort score с прозрачным объяснением
- Отметка отчёта как решённого, подтверждение перед удалением
- Цветовая индикация комфортных/проблемных условий (badge, progress bar)
- Адаптивный дизайн (десктоп/мобильный)
